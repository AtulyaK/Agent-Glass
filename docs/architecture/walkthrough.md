# Agent Glass: Roadmap Implementation Walkthrough

We successfully implemented all four phases of the Agent Glass roadmap, evolving the project from a basic scaffold into a robust, proactive safety system.

## Phase 1: LangGraph Agent
We converted the stub agent into a real LangGraph-powered planner that emits traces:
*   Added `langgraph` dependency and integrated it into `services/agent/app/main.py`.
*   Designed a 4-node graph: `plan_node` → `trace_plan_node` → `simulate_tool_node` → `trace_tool_node`.
*   The agent now takes an objective, asks an NVIDIA LLM to generate a step-by-step plan, simulates the first tool execution (with a 20% failure rate to test roadblocks), and emits `plan_created` and `tool_call` events to the trace gateway.
*   **Database Schema Fix**: Altered the Postgres schema to explicitly size vectors as `VECTOR(1024)` to match the NVIDIA E5 embeddings model, and added `threat_class` and `human_note` columns to `critic_decisions`.

## Phase 2: Smarter Critic
The safety critic service was completely rewritten to support structured evaluations:
*   **Structured Output**: Moved the LLM from unstructured text to a strict JSON schema (`decision`, `flag`, `threat_class`, `rationale`, `halt`).
*   **Threat Classification**: The critic now assigns specific tags (`malicious`, `roadblock`, `loop`, `none`) based on its evaluation of the plan and historical context.
*   **Loop Detection**: Implemented a heuristic pre-check that scans the last 5 decisions; if 4 or more were non-green, the critic forcefully tags the event as a `loop`.
*   **Human-in-the-Loop**: Added a `POST /feedback` endpoint to inject operator notes directly into the next critic prompt for a session.

## Phase 3: Evaluation Pipeline
Enhanced the evaluation infrastructure to prove the critic's effectiveness:
*   **Accuracy Breakdown**: The `evaluator`'s `/run` endpoint now returns accuracy scores grouped by category (e.g., `roadblock` vs `malicious`).
*   **Failure Injection**: Added `POST /inject-failure` to `synthetic-gen`. You can now submit a base prompt and automatically inject specific failure modes (e.g., `tool_unavailable` or `unauthorized_access`) to test the critic's response.
*   **GAIA Benchmark Integration**: Wrote a standalone Python script (`scripts/ingest_gaia.py`) that downloads the GAIA dataset from HuggingFace and ingests tasks into the database as realistic synthetic samples.

## Phase 4: Frontend Dashboard
Built a complete, real-time observability dashboard for monitoring the agent:
*   **SSE Streaming**: Added a `GET /stream` Server-Sent Events endpoint to `trace-gateway` that pushes live event updates to connected browser clients.
*   **Rich UI**: Replaced the frontend stub located in `services/frontend/app/main.py` with a rich HTML/CSS dashboard using a premium dark theme.
*   **Live Metrics**: The dashboard now shows active sessions with color-coded safety flags (Green/Yellow/Red), an expandable live feed of event traces, and a dedicated panel viewing the critic's rationale and threat tags over time.
*   **CORS Fixes**: Added proper CORS middleware to the `trace-gateway`, `critic`, and `agent` services to allow the frontend smooth communication with backend APIs directly from the browser.

## Phase 5: Scientific Evaluation Dataset & 3D Visualization
*   **Dataset Genesis:** Wrote a unified ingestion pipeline to compile `ahsanayub` and `codesagar` malicious prompt datasets alongside the `GAIA` benchmark. The system extracted ~470k samples and cleanly distributed them into stratified `train`/`val`/`test` JSONL sets.
*   **Vector Fix:** Debugged a critical crash in the NVIDIA NIM integration where the `nv-embedqa-e5-v5` embedder rejected traces due to missing the `input_type="query"` requisite for asymmetric models. 
*   **3D UMAP Visualization:** Augmented `trace-gateway` with a `GET /visualization/umap` endpoint wrapping `umap-learn`. This natively projects the 1024-dimensional event embeddings down to 3 dimensions. Integrated **Plotly.js** into the Dashboard to visualize these clusters interactively, mapping coordinate regions to the respective `malicious` / `roadblock` safety flags.

## Phase 6: Robust Batch Embedding System
*   **Token Limit Remediation:** Upgraded the `NVIDIA_EMBED_MODEL` mapping to use `nvidia/nv-embed-v1`, which natively supports a massive **32,000-token context window**.
*   **Vector Dimensionality Alignment:** Dropped and recreated the `embeddings` SQL table with a `VECTOR(4096)` schema to accommodate the richer embedding space generated by `nv-embed-v1`.
*   **Throttled Ingestion:** Wrote `scripts/batch_embedder.py`, a robust Python pipeline that trickles synthetic JSONL samples into the Postgres database. This script safely catches `429 Too Many Requests` API errors, implementing an exponential backoff sequence to flawlessly inject large datasets on NVIDIA NIM's free tier.

## Phase 7: Codebase Productionization
We performed a deep refactor to transform the scaffold into a professional monorepo:
*   **Centralized Config**: Extracted all `os.getenv` duplicated calls from the 7 services into a single `pkg/config/env.py` module.
*   **Monorepo Build Strategy**: Rewrote all `Dockerfile` files and `compose.yaml` to build from the repository root, allowing the shared `pkg` library to be injected into all containers.
*   **Google Style Documentation**: Prepended comprehensive, Google Python Style Guide compliant docstrings to every core service.
*   **Performance Optimization**: Wrapped the heavy UMAP calculations in `asyncio.to_thread` executors within the `trace-gateway` to prevent API event-loop stalling.
*   **Vector Deduplication**: Implemented a `checksum` based caching layer in the `embedder` service to intercept and skip duplicate NIM API calls for repeating prompts, significantly reducing operational waste.

## Phase 8: GitHub Deployment
The project is now fully deployed and secured:
*   **Repository Initialization**: Initialized Git and established the remote origin at `https://github.com/AtulyaK/Agent-Glass.git`.
*   **Security First**: Authored a strict `.gitignore` to ensure `.env` secrets, large `.jsonl` datasets, and virtual environments never leak to the public remote.
*   **README Quickstart**: Completely rewrote the root `README.md` into a "Zero-to-Hero" onboarding manual, ensuring any developer can clone, build, and run an evaluation sequence in minutes.
*   **Clean Documentation**: Organized all architectural artifacts, roadmap logs, and system diagrams into a dedicated `docs/architecture/` directory for long-term project maintainability.

## Phase 9: Hugging Face Dataset Deployment
To save local disk space and finalize the scientific distribution, the entire dataset was migrated to the Hugging Face Hub:
*   **Massive Migration:** Successfully pushed all 470,791 evaluation samples (homogenized from GAIA, ahsanayub, and codesagar) to **`akane69/Agent-Glass`**.
*   **Disk Cleanup**: Reclaimed approximately **800MB** of local storage by deleting the intermediate `.jsonl` data buffers.
*   **Remote Loading**: Updated the ingestion logic so that the system can now pull directly from the Hub, making the evaluation pipeline truly portable and "headless."

Everything is pushed, secured, and ready for scientific evaluation at scale!
